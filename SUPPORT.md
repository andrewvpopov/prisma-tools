# Support

Use [GitHub Issues](https://github.com/andrewpopov/prisma-tools/issues) for
reproducible bugs and scoped feature proposals. Include the package version,
Node version, expected behavior, actual behavior, and a minimal reproduction.

Support is best-effort; no response-time or compatibility SLA is offered.
Questions about a consuming application's deployment or credentials belong with
that application. Security reports must follow [SECURITY.md](./SECURITY.md), not
public issues.
